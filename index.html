About the Website

A case study illustrates a traditional business's transformation into a Distributed Ledger Technology (DLT) business model, reaping the benefits of transparency, increasing trust and empowering users.

  • In a property development business environment, there are various improvements in different stages:
  • Digitalised title management in a property transaction improves the due diligence process, reducing multiple access to the same title by different parties, for example, lawyers, surveyors, estate agents
  • Digitalised construction smart contracts enable smooth stage payments to contractors, thus reducing delays in the completion of the project
  • Tracking the provenance, quantity, movement and location of the building materials for the building site, hence reducing delays and overbudget for construction projects

In the sales of the project – a decentralised property listing connect buyers and estate agents for a speedier property search – digitised title document and identity to improve due diligence for mortgage and conveyancing process – improving timing for completion of sales.

Illustration of blockchain transaction flow

Below is a typical process of blockchain transaction flow

  • 01 The registration of a new user

    A KYC/AML process will be completed before a new user can be accepted to the network.

  • 02 Request transaction

    The new user can request a transaction on the network.

  • 03 Transaction validation

    The transaction is broadcast to a peer-to-peer network consisting of computers known as nodes for validation.

  • 04 Data + Timestamp + Previous hash

    Once the required number of peers verifies the transaction. The data, timestamp and hash from the previous block will form a hash for the current block.

  • 05 New block add to the ledger

    The new block will append to the chain.

  • 06 Broadcast to peers

    This will be broadcast to the network, and peers can view the latest ledger through API.
